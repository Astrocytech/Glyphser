# Glossary

- Profile: A named execution policy that defines capability and environment expectations.
- Fixture: A deterministic input pack used for repeatable verification (for example `hello-core`).
- Evidence: Output artifacts produced by verification (trace, checkpoint, certificate, reports).
- Certificate: A signed-style manifest that binds trace/checkpoint identities for a run.
- Gate: An automated policy check that yields PASS/FAIL for structure, schema, or conformance.
- Determinism Digest: A stable hash derived from canonicalized execution output.
- Conformance: Validation that behavior and artifacts match declared contracts.
