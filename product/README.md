# Product

External-facing documentation, operations runbooks, and site content.

## Contents
- `product/handbook/guides/`: API and contract-facing references.
- `product/handbook/policies/`: lifecycle, support, and governance-facing product policies.
- `product/handbook/reports/`: release and milestone report artifacts/templates.
- `product/handbook/how_to/`: operational and execution runbooks.
- `product/ops/`: operational runbooks.
- `product/site/`: public site content and assets.

## Rule
Product docs should reference canonical `specs/` and `specs/schemas/`, not duplicate normative definitions.
